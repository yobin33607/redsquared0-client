# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository, please report it responsibly.

Preferred reporting methods:
- Use GitHub Security Advisories if available
- Open a private issue on GitHub labeled `security`

When reporting, include:
- A clear description of the issue
- Steps to reproduce the problem
- Any relevant code paths or configuration details
- Suggested mitigations if possible

## Supported Versions

This repository follows the `main` branch as the current supported version. Security fixes are applied there first.

## What to Expect

Once a report is received, maintainers will:
- Acknowledge receipt promptly
- Validate the issue
- Coordinate a fix and release
- Notify the reporter when resolved

## Disclosures

Do not publish security issues publicly until a fix has been released.
