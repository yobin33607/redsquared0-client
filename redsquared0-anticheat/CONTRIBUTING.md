# Contributing to RedSquared0-Anticheat

Thank you for your interest in contributing to this project. This repository is intended to remain a high-quality reference for usermode anti-cheat research and implementation.

## Ways to Contribute

- Report bugs and issues
- Suggest new detections or prevention techniques
- Improve documentation and examples
- Submit code improvements, refactors, and fixes

## Before You Start

1. Read the existing code and documentation.
2. Search open issues and existing pull requests to avoid duplication.
3. Open an issue first if the change is substantial or if you need guidance.

## Development Workflow

- Fork the repository.
- Create a feature branch with a descriptive name.
- Make your changes in a single branch.
- Keep commits focused and well-described.
- Submit a pull request against the `main` branch.

## Pull Request Guidelines

A good pull request should include:
- A clear summary of the change
- The problem being addressed
- Testing performed
- Any relevant references or issue links

Please do not merge your own pull request without prior maintainer approval.

## Coding Standards

- Follow the existing C++ style in the repository.
- Keep formatting consistent with surrounding code.
- Prefer clear, readable implementations over clever hacks.
- Add comments for non-obvious logic or security-sensitive behavior.

## Building and Testing

This project supports Visual Studio and CMake builds.

- Build the appropriate solution or generate a CMake configuration.
- Verify your change compiles cleanly.
- Test relevant functionality locally.

## Documentation

If your change affects usage, configuration, or behavior, update the relevant documentation.
- `README.md`
- `changelog.md`
- Wiki pages if applicable

## License

By contributing, you agree that your contributions will be licensed under the MIT license of this repository.

## Security-Sensitive Contributions

If you believe you have found a security vulnerability, please do not disclose it publicly. Use the security policy in `SECURITY.md` to report it privately.
